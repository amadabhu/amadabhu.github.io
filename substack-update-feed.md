# Refreshing the Substack feed

`_includes/substack-feed.html` is generated by
[scripts/build_substack_feed.py](scripts/build_substack_feed.py). It used to
run on a GitHub Actions cron, but Substack 403s requests from GitHub's
datacenter IPs, so the refresh is now local-only.

## When to run

After publishing a new post on
[achyuthhacks.substack.com](https://achyuthhacks.substack.com), regenerate
the feed and commit so the home page reflects it.

## How to run

From the repo root:

```bash
python3 scripts/build_substack_feed.py
```

The script uses only the Python stdlib — no venv or `pip install` needed.
It overwrites `_includes/substack-feed.html` with the latest 5 entries.

## Commit and push

```bash
git add _includes/substack-feed.html
git commit -m "chore(substack): refresh feed"
git push
```

GitHub Pages picks up the change on the next deploy.

## Why not Actions

`scripts/build_substack_feed.py` fetches `https://achyuthhacks.substack.com/feed`
with `urllib`. From a residential IP that returns the RSS feed; from a GitHub
Actions runner Substack returns `HTTP 403 Forbidden`. The User-Agent header
doesn't change that — Substack blocks based on source IP range. Working around
it would mean a browser-like UA + cookies, or a residential proxy, neither of
which is worth the maintenance for a site that updates by hand anyway.
